## ğŸ”Œ Border Router Selection

### Thread Border Routers

A **Thread Border Router (TBR)** connects the Thread mesh network to your IP network (WiFi/Ethernet).

| Border Router | Platform | Price | Note |
|---------------|----------|-------|------|
| **HomePod Mini** | Apple | ~$99 | Best with iPhone |
| **Apple TV 4K** | Apple | ~$129 | With Ethernet |
| **Google Nest Hub (2nd gen)** | Google | ~$99 | Best with Android |
| **Google Nest WiFi Pro** | Google | ~$199 | Router + TBR |
| **HA SkyConnect** | Home Assistant | ~$30 | Requires OTBR add-on |
| **HA Yellow** | Home Assistant | ~$125 | Built-in radio |
| **SONOFF ZBDongle-E** | DIY | ~$20 | With Thread firmware |
| **SMLIGHT SLZB-06** | DIY | ~$50 | PoE + Thread |

<Aside type="tip" title="Recommendation">
**iPhone users:** Use HomePod Mini/Apple TV as Thread Border Router.

**Android users:** Use Google Nest Hub (2nd gen) as Thread Border Router.

**DIY/Advanced:** Use HA SkyConnect + OpenThread Border Router add-on.
</Aside>

## ğŸ“¦ Installation in Home Assistant

### Method 1: Existing Border Router (Easiest)

If you have HomePod Mini, Apple TV 4K, or Google Nest Hub:

<Steps>
1. Install **Thread integration**:
   - **Settings â†’ Devices & Services â†’ Add Integration**
   - Search "Thread"

2. Install **Matter integration**:
   - **Settings â†’ Devices & Services â†’ Add Integration**
   - Search "Matter"
   - Check "Use the official Matter Server Supervisor add-on"

3. Configure Thread credentials:
   - Go to **Thread integration**
   - Find your border router
   - Click **"Add to preferred network"**
   - Click **"Use router for Android + iOS credentials"**

4. Restart Home Assistant
</Steps>

### Method 2: OpenThread Border Router (Advanced)

For HA SkyConnect, Yellow, or third-party dongles:

<Tabs>
  <TabItem label="HA Add-on">
    <Steps>
    1. Flash Thread firmware on your dongle:
       - Go to [SkyConnect Web Flasher](https://skyconnect.home-assistant.io/)
       - Select "Thread" firmware
    
    2. Install **OpenThread Border Router** add-on:
       - **Settings â†’ Add-ons â†’ Add-on Store**
       - Search "OpenThread Border Router"
       - Install
    
    3. Configure add-on:
       ```yaml
       device: /dev/serial/by-id/usb-Nabu_Casa_SkyConnect...
       baudrate: 460800
       flow_control: true
       autoflash_firmware: true
       ```
    
    4. Start add-on and enable:
       - "Start on boot"
       - "Watchdog"
    
    5. Install integrations:
       - **Thread** integration
       - **OpenThread Border Router** integration
       - **Matter** integration
    
    6. Configure preferred network (see Method 1, step 3)
    </Steps>
  </TabItem>

## â• Add Matter Devices

### Via Home Assistant App (Recommended)

<Steps>
1. Open **Home Assistant app** on phone

2. Go to **Settings â†’ Devices & Services**

3. Select **Devices** tab â†’ **Add device**

4. Select **Add Matter device**

5. Select **"No, it's new"** (for new device)

6. **Scan QR code** on the device
   - Or enter manual pairing code

7. Wait for commissioning (can take 1-5 minutes)

8. Name the device
</Steps>

<Aside type="caution" title="Commissioning Requirements">
**For Thread devices:**
- Border router must be on the same network
- Phone must be close to border router AND the device

**For WiFi Matter devices:**
- Phone must be on 2.4 GHz WiFi
- Same network as Home Assistant
</Aside>

### Via Share Sheet (Multi-admin)

If the device is already added to Google/Apple:

<Steps>
1. In Google Home/Apple Home, find the device

2. Select **"Share device"** or **"Turn on pairing mode"**

3. Share with Home Assistant app

4. Device is added as additional controller
</Steps>

## ğŸ” Troubleshooting

### Device Cannot Pair

<Steps>
1. **Check border router is online:**
   - Thread integration â†’ Check status

2. **Factory reset the device:**
   - See device manual
   - Typically: Hold button 10+ seconds

3. **Move closer:**
   - Phone close to border router
   - Device close to border router

4. **Check WiFi (for WiFi Matter):**
   - Must be 2.4 GHz
   - Same network as HA

5. **Restart:**
   - Home Assistant
   - Border router
   - Matter Server add-on
</Steps>

### Device Offline

```yaml
# Typical causes:
1. Thread network not available
2. Border router offline
3. Device too far from mesh

# Solutions:
- Add more Thread routers (any powered Thread device)
- Move border router centrally
- Check Thread network map in HA
```

| Feature | Thread/Matter | Zigbee | Z-Wave |
|---------|---------------|--------|--------|
| **IP-native** | âœ… | âŒ | âŒ |

<Aside type="tip" title="When to Choose What?">
**Choose Matter/Thread if:**
- You want to use devices with multiple systems
- You're starting from scratch
- You prioritize future-proofing

**Stay with Zigbee/Z-Wave if:**
- You have existing setup
- You need specific devices
- You want most choice now
</Aside>
