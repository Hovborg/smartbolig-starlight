### Where to Find Logs?

| Log Type | Location |
|----------|----------|
| **Core logs** | Settings â†’ System â†’ Logs |
| **Supervisor logs** | Settings â†’ System â†’ Logs â†’ Supervisor |
| **Add-on logs** | Settings â†’ Add-ons â†’ [Add-on] â†’ Log |
| **Full log file** | `/config/home-assistant.log` |

### Enable Debug Logging

```yaml
# configuration.yaml
logger:
  default: info
  logs:
    # Enable debug for specific component
    homeassistant.components.zha: debug
    homeassistant.components.mqtt: debug
    # Or a specific integration
    custom_components.hacs: debug
```

<Aside type="tip" title="Remember to disable debug">
Debug logging generates A LOT of data and can fill your disk. Turn it off again when the problem is solved.
</Aside>

## ğŸš« Home Assistant Won't Start

### Symptom: Blank screen or "Unable to connect"

<Steps>
1. **Wait 5-10 minutes** - Startup can take time

2. **Check if HA is running:**
   ```bash
   # SSH into your server
   ha core info
   ```

3. **View startup logs:**
   ```bash
   ha core logs
   ```

4. **Common fixes:**
   - Check configuration.yaml for syntax errors
   - Remove recently added integrations
   - Boot in safe mode
</Steps>

### Boot in Safe Mode

```bash
# Via SSH or terminal
ha core restart --safe-mode
```

Safe mode starts HA without custom integrations and add-ons.

### Configuration Errors

```yaml
# Validate configuration BEFORE restart
# Settings â†’ System â†’ Developer Tools â†’ YAML â†’ Check Configuration

# Or via command line:
ha core check
```

## ğŸ“¡ Zigbee Problems

### Device Falls Off Network

| Cause | Solution |
|-------|----------|
| **Weak signal** | Add more routers (powered devices) |
| **Interference** | Move coordinator away from USB 3.0 / WiFi router |
| **Firmware** | Update coordinator firmware |
| **Crowded channel** | Change Zigbee channel (11, 15, 20, 25) |

### Pairing Fails

<Steps>
1. **Put device in pairing mode** (see manual)

2. **Ensure coordinator is ready:**
   - Zigbee2MQTT: Permit Join enabled
   - ZHA: Add device enabled

3. **Place device CLOSE to coordinator** during pairing

4. **Factory reset device** and try again

5. **Check device is supported:**
   - [Zigbee2MQTT Supported Devices](https://www.zigbee2mqtt.io/supported-devices/)
   - [ZHA Device Support](https://zigpy.github.io/)
</Steps>

### USB Coordinator Disappears

```bash
# Find USB device
ls -la /dev/serial/by-id/

# Use the permanent device name in configuration:
# /dev/serial/by-id/usb-Silicon_Labs_CP2102-xxxxx
# NOT /dev/ttyUSB0 (can change)
```

## ğŸ“¶ WiFi Devices

### Shelly Not Showing

| Check | Solution |
|-------|----------|
| **Same network** | Device must be on same subnet |
| **mDNS** | Enable mDNS on your router |
| **IP address** | Find IP via router or Shelly app |
| **Firewall** | Allow UDP port 5683 (CoAP) |

### Tasmota/ESPHome Offline

```yaml
# Check WiFi signal strength
# Entity: sensor.device_wifi_signal

# If signal is weak:
# 1. Move device closer to AP
# 2. Add WiFi repeater
# 3. Switch to 2.4 GHz (better range)
```

## ğŸ”„ Automations Not Working

### Debugging Checklist

<Steps>
1. **Is automation enabled?**
   - Check toggle in Automations list

2. **Trigger correct?**
   - Settings â†’ Automations â†’ [Automation] â†’ Trace
   - Check "Last triggered"

3. **Conditions met?**
   - Test each condition individually
   - Use Developer Tools â†’ States

4. **Actions failing?**
   - Check logs for errors
   - Test action manually via Services
</Steps>

### Test Automation Manually

```yaml
# Developer Tools â†’ Services
service: automation.trigger
target:
  entity_id: automation.your_automation
data:
  skip_condition: true  # Skip conditions
```

### Trace Automation

<Steps>
1. Go to **Settings â†’ Automations**

2. Click on the automation

3. Click **Traces** (trace icon)

4. See exactly what happened on last run:
   - Which triggers activated
   - Which conditions were true/false
   - Which actions ran/failed
</Steps>

## ğŸ—„ï¸ Database Problems

### "Database is locked"

```bash
# Stop Home Assistant
ha core stop

# Wait 30 seconds

# Start again
ha core start
```

### Corrupt Database

```bash
# Backup first!
cp /config/home-assistant_v2.db /config/home-assistant_v2.db.backup

# Delete database (loses history)
rm /config/home-assistant_v2.db

# Restart - new database created automatically
ha core restart
```

### Database Too Large

See [Performance Guide](/en/home-assistant/performance/) for database optimization.

## ğŸ”Œ Integration Problems

### Integration Won't Load

| Problem | Solution |
|---------|----------|
| **Missing dependencies** | Restart HA, check logs |
| **Incompatible version** | Update integration |
| **Configuration error** | Check integration docs |
| **Credentials expired** | Re-authenticate |

### Re-authenticate Integration

<Steps>
1. Go to **Settings â†’ Devices & Services**

2. Find the integration

3. Click **Configure** or **Re-authenticate**

4. Follow OAuth flow again
</Steps>

### HACS Integration Problems

```bash
# Reinstall HACS integration
1. Remove integration from Devices & Services
2. Delete folder: /config/custom_components/[integration]
3. Restart HA
4. Install again via HACS
```

## ğŸ” Login Problems

### Forgot Password

```bash
# Via command line
ha auth reset --username YOUR_USERNAME --password NEW_PASSWORD
```

### 2FA Lost

```bash
# Remove TOTP for user
ha auth totp remove --username YOUR_USERNAME
```

### Locked Out

```yaml
# Add temporarily to configuration.yaml:
homeassistant:
  auth_providers:
    - type: trusted_networks
      trusted_networks:
        - 192.168.1.0/24
      allow_bypass_login: true
    - type: homeassistant

# Restart, log in, remove again
```
