### 2. Claude Code releases: v2.1.211

**What changed:** Claude Code releases published the update on 15 Jul 2026. Claude Code v2.1.211 adds a --forward-subagent-text flag &#40;and matching environment variable&#41; to include subagent text and thinking in stream-json output. It also fixes permission previews sent to chat channels: bidirectional-override, zero-width, and look-alike quote characters are now neutralized so tool inputs cannot visually alter the approval message shown to a reviewer. A separate fix stops auto mode from overriding a PreToolUse hook's ask decision for unsandboxed Bash commands.

> Short source excerpt: What's changed Added --forward-subagent-text flag and CLAUDE&#95;CODE&#95;FORWARD&#95;SUBAGENT&#95;TEXT environment variable to include subagent text and thinking in stream-json output Fixed permission…

**Why it matters:** The permission-preview fix matters most: without it, a malicious tool input could visually disguise what a reviewer is approving in a chat-based workflow, so this closes a real spoofing risk. The hook fix means an "ask" rule you set for unsandboxed Bash will actually pause for confirmation instead of being silently auto-approved, restoring control over risky commands run by an agent.

**How to verify it:** If you use chat-relayed approvals or Bash hooks with ask rules, update to v2.1.211 and re-check the release notes on the Claude Code GitHub page to confirm which hook and channel behaviors changed before trusting existing approval workflows.
