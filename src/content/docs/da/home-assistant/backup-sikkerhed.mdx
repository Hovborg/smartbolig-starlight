## 🆕 Automatisk Backup (2025.1+)

Home Assistant 2025.1 introducerede et **helt nyt backup-system**:

### Nye Features

| Feature | Beskrivelse |
|---------|-------------|
| **Automatiske backups** | Planlagte backups uden add-ons |
| **Flere lokationer** | Cloud + NAS + lokal samtidig |
| **Kryptering** | Standard for cloud, valgfri for lokal |
| **Retention policies** | Automatisk sletning af gamle backups |
| **Nem opsætning** | Wizard til hurtig konfiguration |

### Opsæt Automatisk Backup

<Steps>
1. Gå til **Indstillinger → System → Backups**

2. Klik **Konfigurer automatiske backups**

3. Vælg tidsplan:
   - Dagligt, ugentligt, eller efter opdateringer
   - Tidspunkt (f.eks. kl. 04:00)

4. Vælg retention:
   - Antal backups der gemmes (f.eks. 3)

5. Vælg lokationer:
   - Lokal
   - Google Drive
   - OneDrive
   - Home Assistant Cloud
   - NAS (SMB/NFS)

6. Vælg kryptering:
   - **GEM DIN KRYPTERINGSNØGLE!** (Password manager)

7. Klik **Gem**
</Steps>

<Aside type="danger" title="Vigtigt!">
Hvis du mister din krypteringsnøgle, kan du **ikke** gendanne dine backups. Gem nøglen et sikkert sted - gerne flere steder.
</Aside>

## ☁️ Cloud Backup Lokationer

### Google Drive (Native 2025.2+)

<Steps>
1. Gå til **Indstillinger → Enheder & Tjenester**

2. Klik **Tilføj Integration** → søg "Google Drive"

3. Følg Google OAuth flow:
   - Log ind på din Google-konto
   - Giv adgang til Google Drive

4. Gå til **Indstillinger → System → Backups → Lokationer**

5. Tilføj Google Drive som backup-lokation

6. Backups gemmes automatisk i mappe "Home Assistant"
</Steps>

**Gratis lagerplads:** 15 GB (delt med Gmail og Photos)

### OneDrive (Native 2025.2+)

Samme proces som Google Drive - brug OneDrive integrationen.

### Home Assistant Cloud (Nabu Casa)

Hvis du abonnerer på Home Assistant Cloud:

<Steps>
1. Gå til **Indstillinger → Home Assistant Cloud**

2. Cloud backup er automatisk tilgængelig

3. Backups er **altid krypterede** (kan ikke slås fra)

4. Tilføj som backup-lokation under Backups
</Steps>

**Pris:** ~€7.50/måned (inkluderer fjernaccess, Alexa/Google integration)

## 🗄️ NAS Backup (SMB/NFS)

### Forbered NAS

<Tabs>
  <TabItem label="Synology">
    <Steps>
    1. Åbn **Control Panel → User & Group**
    
    2. Opret bruger til Home Assistant:
       - Brugernavn: `homeassistant`
       - Stærkt password
       - Deaktiver adgang til DSM
    
    3. Åbn **Control Panel → Shared Folder**
    
    4. Opret delt mappe:
       - Navn: `ha-backups`
       - Giv HA-bruger fuld adgang
    
    5. Aktiver SMB under **File Services → SMB**
    </Steps>
  </TabItem>
  <TabItem label="TrueNAS/QNAP">
    Lignende proces:
    - Opret dedikeret bruger
    - Opret dedikeret share
    - Sæt rettighedertil read/write
    - Aktiver SMB/NFS
  </TabItem>
</Tabs>

### Tilføj NAS til Home Assistant

<Steps>
1. Gå til **Indstillinger → System → Storage**

2. Klik **Tilføj Netværkslagring**

3. Udfyld:
   ```text
   Navn: NAS Backup
   Brug: Backup
   Server: 192.168.1.50  # Din NAS IP
   Type: SMB (eller NFS)
   Share: ha-backups
   Brugernavn: homeassistant
   Password: ****
   ```

4. Klik **Forbind**

5. Gå til **Backups → Lokationer** og tilføj NAS
</Steps>

## 🔐 Kryptering

### Hvornår Kryptere?

| Lokation | Anbefaling | Grund |
|----------|------------|-------|
| **Cloud** | ✅ Altid | Data forlader dit netværk |
| **Home Assistant Cloud** | 🔒 Automatisk | Kan ikke slås fra |
| **NAS (hjemme)** | ⚖️ Valgfrit | Lettere recovery uden |
| **USB-disk** | ✅ Ja | Kan mistes/stjæles |

### Krypteringsnøgle Håndtering

```text
# GEM DISSE STEDER:
1. Password Manager (1Password, Bitwarden, etc.)
2. Fysisk kopi i pengeskab/sikkerhedsboks
3. Delt med en betroet person

# GEM IKKE:
- I Home Assistant selv (mistes ved crash)
- I ukrypteret fil på NAS
- Kun i hovedet
```

## 📅 Backup Strategi

### Anbefalet Setup

| Backup Type | Frekvens | Retention | Lokation |
|-------------|----------|-----------|----------|
| **Daglig** | Hver nat kl. 04:00 | 3 stk | Lokal |
| **Ugentlig** | Søndag kl. 04:00 | 4 stk | NAS + Cloud |
| **Før opdatering** | Ved hver HA opdatering | 3 stk | Alle lokationer |

### Konfigurer Differentierede Backups

```yaml
# Automatisk backup konfiguration
# Indstillinger → System → Backups → Automatisk backup

Tidsplan:
  Frekvens: Daglig
  Tid: 04:00
  Dag: Alle dage

Retention:
  Antal kopier: 3

Lokationer:
  - Lokal (krypteret: nej)
  - Google Drive (krypteret: ja)
  - NAS (krypteret: nej)

Ved opdateringer:
  Opret backup: Ja
```

## 🔄 Gendannelse (Restore)

### Fra Backup UI

<Steps>
1. Gå til **Indstillinger → System → Backups**

2. Find backup (fra lokal, cloud, eller NAS)

3. Klik på backup → **Gendan**

4. Vælg hvad der skal gendannes:
   - Alt
   - Kun konfiguration
   - Specifikke add-ons

5. Vent på gendannelse (kan tage 5-30 min)

6. Log ind igen efter genstart
</Steps>

### Disaster Recovery (Ny Installation)

Hvis din HA-server er totalt død:

<Steps>
1. **Installer Home Assistant** på ny hardware

2. Kør gennem initial setup

3. **Monter backup-lokation:**
   - NAS: Tilføj netværkslagring
   - Google Drive: Installer integration
   - USB: Tilslut disk

4. Gå til **Indstillinger → System → Backups**

5. Find backup fra din lokation

6. **Gendan fuld backup**

7. Genstart og log ind med dine gamle credentials
</Steps>

<Aside type="tip" title="Cross-platform restore">
Home Assistant backups virker på tværs af CPU-arkitekturer! Du kan gendanne en backup fra Raspberry Pi på en x86 server og omvendt.
</Aside>

## 🛡️ Sikkerhed Best Practices

### Netværkssikkerhed

| Praksis | Implementering |
|---------|----------------|
| **Separat VLAN** | IoT-enheder i eget netværk |
| **Firewall** | Blokér unødvendig trafik |
| **Ingen port forwarding** | Brug VPN eller Nabu Casa |
| **HTTPS** | SSL/TLS på al trafik |

### Home Assistant Sikkerhed

```yaml
# configuration.yaml
http:
  ssl_certificate: /ssl/fullchain.pem
  ssl_key: /ssl/privkey.pem
  ip_ban_enabled: true
  login_attempts_threshold: 5
```

### Bruger Sikkerhed

| Praksis | Beskrivelse |
|---------|-------------|
| **Stærke passwords** | 16+ tegn, password manager |
| **2FA** | Aktiver for alle brugere |
| **Least privilege** | Giv kun nødvendige rettigheder |
| **Separate konti** | Én per husstand, admin separat |

### Aktiver 2FA

<Steps>
1. Gå til dit **Profilbillede** (nederst til venstre)

2. Scroll til **Multifactor Authentication Modules**

3. Aktiver **Time-based One-Time Password**

4. Scan QR-kode med authenticator app:
   - Google Authenticator
   - Authy
   - 1Password

5. Indtast kode for at bekræfte
</Steps>

## 🔧 Fejlfinding

### Backup Fejler

```yaml
# Tjek:
1. Er der nok plads på destinations-disk?
2. Er netværksforbindelse stabil (NAS)?
3. Er credentials korrekte (cloud/NAS)?
4. Check Supervisor logs for detaljer
```

### Gendannelse Fejler

```yaml
# Tjek:
1. Er backup'en korrupt? (prøv anden backup)
2. Er krypteringsnøgle korrekt?
3. Er HA version kompatibel?
4. Har du nok diskplads?
```

### NAS Mount Fejler

```yaml
# Almindelige problemer:
1. SMB version mismatch → Prøv SMB 2.0 eller 3.0
2. Forkerte credentials → Tjek brugernavn/password
3. Share eksisterer ikke → Tjek share-navn på NAS
4. Firewall blokerer → Port 445 (SMB) skal være åben
```
