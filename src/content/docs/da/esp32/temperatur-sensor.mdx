En temperatur-/fugtighedssensor er et godt sted at starte: tre ledninger, en håndfuld linjer YAML, og du har live-målinger fra rummet i Home Assistant. Komponenterne koster omkring 70 kr.

## 📋 Forudsætninger

Før du starter:

- ✅ [ESP32 sat op med ESPHome](/da/esp32/kom-godt-i-gang/)
- ✅ Home Assistant kørende

<Aside type="note">
Ny til ESP32? Start med vores [Kom godt i gang guide](/da/esp32/kom-godt-i-gang/) først.
</Aside>

## 🛒 Komponenter

| Del | Beskrivelse | Pris ca. |
|-----|-------------|----------|
| ESP32 DevKit | WiFi mikrocontroller | 45 kr |
| DHT22 sensor | Temperatur + fugtighed | 25 kr |
| Jumper wires | 3 stk han-hun | 10 kr |
| **Total** | | **~70 kr** |

<Aside type="tip" title="DHT22 vs DHT11">
- **DHT22:** ±0.5°C præcision, 0-100% fugtighed - **anbefalet**
- **DHT11:** ±2°C præcision, 20-80% fugtighed - billigere men upræcis
</Aside>

## 🔌 Tilslutning

### Wiring diagram

![DHT22 til ESP32 wiring diagram](/images/guides/temperatur-sensor/wiring.svg)

### Trin-for-trin

<Steps>
1. **Rød ledning (VCC/+)** → ESP32 **3.3V** pin

2. **Gul/hvid ledning (DATA/OUT)** → ESP32 **GPIO4** pin

3. **Sort ledning (GND/-)** → ESP32 **GND** pin
</Steps>

<Aside type="note" title="Pull-up-modstand">
Bruger du en nøgen 4-bens DHT22, skal der sidde en 10 kΩ modstand (pull-up) mellem DATA og 3.3V for stabile aflæsninger. De fleste 3-bens moduler på breakout-print har modstanden indbygget, så der rækker de tre ledninger.
</Aside>

<Aside type="danger" title="Forsyn sensoren fra 3.3V">
DHT22 tåler i sig selv 3,3-5,5 V, men ESP32'ens GPIO-ben er **ikke 5V-tolerante**. Forsyner du sensoren fra 5V, følger datalinjen med op og kan beskadige ESP32'en. Brug derfor 3.3V-benet.
</Aside>

## ⚙️ ESPHome Konfiguration

Åbn din enhed i ESPHome og tilføj denne sensor-konfiguration:

```yaml title="stue-sensor.yaml" {15-24}
esphome:
  name: stue-sensor
  friendly_name: Stue Sensor

esp32:
  board: esp32dev

logger:
api:
  encryption:
    key: "din-nøgle-her"
ota:
  platform: esphome

# DHT22 Sensor
sensor:
  - platform: dht
    pin: GPIO4
    model: DHT22
    temperature:
      name: "Stue Temperatur"
      filters:
        - offset: 0.0  # Juster hvis nødvendigt
    humidity:
      name: "Stue Fugtighed"
    update_interval: 60s

wifi:
  ssid: "dit-wifi"
  password: "din-kode"
```

<Aside type="tip" title="Kalibrering">
Hvis temperaturen viser forkert, så sammenlign med et termometer du stoler på, og juster `offset`-værdien. F.eks. trækker `offset: -1.5` 1.5°C fra målingen, mens en positiv værdi lægger til.
</Aside>

### Alternativ: DS18B20 (vandtæt)

Skal du måle udendørs eller i væske, er en vandtæt DS18B20 et bedre valg end DHT22. Den bruger 1-wire og kræver en 4,7 kΩ pull-up-modstand mellem DATA og 3.3V:

```yaml title="DS18B20 i stedet for DHT22"
one_wire:
  - platform: gpio
    pin: GPIO4

sensor:
  - platform: dallas_temp
    name: "Stue Temperatur"
    update_interval: 60s
```

Med kun én sensor på bussen behøver du ikke angive `address`. Bemærk at DS18B20 kun måler temperatur, ikke fugtighed.

## ✅ Verificer i Home Assistant

<Steps>
1. Gå til **Indstillinger** → **Enheder og tjenester**

2. Find din ESP32 enhed og klik på den

3. Du bør se to nye sensorer:
   - `sensor.stue_temperatur`
   - `sensor.stue_fugtighed`

4. Klik på en sensor for at se graf over tid
</Steps>

## 📊 Tilføj til Dashboard

### Simpelt sensor-kort

```yaml title="Lovelace kort"
type: sensor
entity: sensor.stue_temperatur
name: Stue
icon: mdi:thermometer
graph: line
hours_to_show: 24
```

### Kombineret temperatur/fugtighed

```yaml title="Lovelace kort"
type: glance
entities:
  - entity: sensor.stue_temperatur
    name: Temperatur
  - entity: sensor.stue_fugtighed
    name: Fugtighed
title: Stue Klima
```

## 🔧 Fejlfinding

<Aside type="tip" title="Sensor viser 'Unavailable'?">
**Tjek disse ting:**
1. Er ledningerne sat rigtigt i? (VCC→3.3V)
2. Er GPIO-pin korrekt i konfigurationen?
3. Nøgen sensor: sidder 10 kΩ pull-up-modstanden mellem DATA og 3.3V?
4. Prøv en anden GPIO-pin (f.eks. GPIO5)
5. Tjek ESPHome-loggen for fejlbeskeder
</Aside>

<Aside type="tip" title="Temperatur virker forkert?">
**Mulige årsager:**
- DHT22 har ±0.5°C tolerance fra fabrikken
- Sensor placeret for tæt på ESP32 (selv-opvarmning)
- Direkte sollys eller nær varmekilde

**Løsning:**
- Brug `offset` filter til kalibrering
- Flyt sensor væk fra ESP32 med længere ledninger
- Sammenlign med et kendt termometer
</Aside>

<Aside type="tip" title="Fugtighed virker forkert?">
DHT22 fugtighed er ofte 5-10% off. Kalibrér med:
```yaml
humidity:
  name: "Stue Fugtighed"
  filters:
    - offset: -5.0  # Træk 5% fra
```
</Aside>

## 💡 Tips til præcise målinger

| Tip | Forklaring |
|-----|------------|
| **Placering** | Undgå direkte sollys, radiatorer og vinduer |
| **Højde** | Placer i ~1.5m højde for rum-temperatur |
| **Luftcirkulation** | Ikke i hjørner eller bag møbler |
| **Opdateringsinterval** | 60s er fint, DHT22 kan højst aflæses hvert 2. sekund |

## 🤖 Automationsidéer

```yaml title="Eksempel: Advar ved høj fugtighed"
automation:
  - alias: "Alarm ved høj fugtighed"
    trigger:
      - platform: numeric_state
        entity_id: sensor.stue_fugtighed
        above: 70
        for: "00:30:00"
    action:
      - service: notify.mobile_app_din_telefon  # udskift med din egen enheds notify-tjeneste
        data:
          title: "⚠️ Høj fugtighed"
          message: "Fugtigheden i stuen er {{ states('sensor.stue_fugtighed') }}%"
```

Udskift `notify.mobile_app_din_telefon` med navnet på din egen telefons notify-tjeneste - du finder det under **Udviklerværktøjer** → **Handlinger**.
