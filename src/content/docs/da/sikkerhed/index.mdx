Smart home betyder også ansvar. Et usikkert setup kan give hackere adgang til dit netværk, dine kameraer og dine data. Tag sikkerhed alvorligt fra starten.

<Aside type="danger" title="⚠️ Sikkerhed er ikke valgfrit">
Usikre smart home enheder er blandt de mest hackede enheder på internettet. Mange enheder har standard passwords, ingen opdateringer, og usikker kommunikation.
</Aside>

## 🔒 Grundlæggende sikkerhed

### ⭐⭐⭐ Kritisk (gør dette først!)

| Handling | Hvorfor | Hvordan |
|----------|---------|---------|
| **Stærke passwords** | Svage passwords = nem adgang | Brug password manager, 16+ tegn |
| **To-faktor på HA** | Ekstra beskyttelse | Din profil → Sikkerhed → Multifaktor |
| **Opdater alt** | Sikkerhedshuller lukkes | Tjek opdateringer ugentligt |
| **Skift standard passwords** | Alle kender dem | Router, kameraer, alt! |

### ⭐⭐ Anbefalet

| Handling | Hvorfor | Hvordan |
|----------|---------|---------|
| **Separat IoT netværk** | Isolerer enheder | VLAN eller gæste-WiFi |
| **Lokal DNS (Pi-hole)** | Blokerer tracking | Kør Pi-hole eller AdGuard |
| **VPN til fjernadgang** | Sikker adgang udefra | WireGuard, Tailscale |
| **Undgå cloud-enheder** | Mindre angrebsflade | Vælg lokale alternativer |

### ⭐ Avanceret

| Handling | Hvorfor | Hvordan |
|----------|---------|---------|
| **Firewall regler** | Kontrollér trafik | pfSense, OPNsense |
| **IDS/IPS** | Opdager angreb | Suricata, Snort |
| **Log analyse** | Opdager misbrug | Graylog, Grafana |

## 🏠 Hvorfor lokalt?

Home Assistant og ESPHome kører **lokalt** på dit netværk. Det er en STOR sikkerhedsfordel:

| Lokalt | Cloud |
|--------|-------|
| ✅ Data forbliver hjemme | ❌ Data sendes til servere |
| ✅ Virker uden internet | ❌ Afhængig af internet |
| ✅ Ingen månedlig betaling | ❌ Ofte abonnement |
| ✅ Du har fuld kontrol | ❌ Producent kan lukke |
| ✅ Hurtigere respons | ❌ Forsinkelse via cloud |

<Aside type="tip" title="Lokal først">
Vælg altid lokale løsninger når det er muligt. Zigbee, Z-Wave og WiFi-enheder med lokal API (Tasmota, ESPHome) er bedre end cloud-afhængige enheder.
</Aside>

## 🌐 Netværkssegmentering

Hold IoT-enheder på et separat netværk:

```
┌─────────────────────────────────────────────────────┐
│                     ROUTER                          │
└─────────────────────────────────────────────────────┘
        │                    │                  │
   ┌────┴────┐         ┌────┴────┐        ┌────┴────┐
   │  VLAN 1 │         │  VLAN 2 │        │  VLAN 3 │
   │ Primært │         │   IoT   │        │  Gæst   │
   │         │         │         │        │         │
   │ PC      │         │ Sensorer│        │ Gæster  │
   │ Telefon │   ──►   │ Kameraer│        │         │
   │ HA      │         │ Pærer   │        │         │
   └─────────┘         └─────────┘        └─────────┘
```

<Aside type="note" title="Simpel løsning">
Har du ikke VLAN-support? Brug din routers **gæstenetværk** til IoT-enheder. Det giver basal isolation.
</Aside>

### Gode passwords

```
❌ Dårligt:   password123

❌ Dårligt:   qwerty
✅ Godt:      korrekt-hest-batteri-hæfteklamme
✅ Godt:      Xk9#mP2$vL8@nQ4&
```

### Brug password manager

| Manager | Pris | Kommentar |
|---------|------|-----------|
| **Bitwarden** | Gratis | ⭐ Anbefalet, open source |
| 1Password | 3$/md | Poleret, familiedeling |
| KeePass | Gratis | Offline, fuld kontrol |

## 🌍 Sikker fjernadgang

Vil du tilgå Home Assistant udefra? Gør det sikkert:

| Metode | Sikkerhed | Sværhedsgrad |
|--------|-----------|--------------|
| **Nabu Casa** | ⭐⭐⭐⭐⭐ | Let |
| **Tailscale VPN** | ⭐⭐⭐⭐⭐ | Let |
| **WireGuard VPN** | ⭐⭐⭐⭐⭐ | Medium |
| Reverse proxy + cert | ⭐⭐⭐⭐ | Svær |
| Port forwarding | ⭐ | **UNDGÅ!** |

<Aside type="danger" title="Aldrig port forward!">
Port forwarding af Home Assistant direkte til internettet er **ekstremt farligt**. Brug VPN eller Nabu Casa i stedet.
</Aside>

## 📋 Sikkerhedstjekliste

### Ved installation

- [ ] Skift alle standard passwords
- [ ] Opdater firmware på alle enheder
- [ ] Aktiver to-faktor på Home Assistant
- [ ] Brug HTTPS (automatisk med Nabu Casa)

### Månedligt

- [ ] Tjek for opdateringer
- [ ] Gennemgå aktive brugere
- [ ] Tjek logs for mærkeligheder
- [ ] Backup af konfiguration

### Ved nye enheder

- [ ] Undersøg producentens sikkerhedspraksis
- [ ] Foretræk enheder uden cloud
- [ ] Opdater firmware før brug
- [ ] Placer på IoT-netværk
