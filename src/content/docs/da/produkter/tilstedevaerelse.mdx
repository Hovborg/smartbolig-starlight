## 🔑 PIR vs mmWave - Hvad Er Forskellen?

### PIR (Passive Infrared)

Traditionelle bevægelsessensorer der detekterer **varmeændringer**:

| Fordele | Ulemper |
|---------|---------|
| ✅ Billig | ❌ Kun bevægelse, ikke stilstand |
| ✅ Lavt strømforbrug | ❌ Kan ikke se gennem glas |
| ✅ Simpel | ❌ Falske triggers fra varmekilder |
| ✅ Pålidelig til bevægelse | ❌ Dårlig til kontor/sofabrug |

**Bedst til:** Entre, trapper, gangarealer - steder hvor du bare passerer igennem.

### mmWave (Millimeter Wave Radar)

Moderne radarsensorer der detekterer **selv minimal bevægelse** (vejrtrækning):

| Fordele | Ulemper |
|---------|---------|
| ✅ Detekterer stillesiddende personer | ❌ Dyrere |
| ✅ Virker gennem tynde vægge | ❌ Kan trigge på kæledyr |
| ✅ Zone-detektion mulig | ❌ Kræver mere opsætning |
| ✅ Afstandsmåling | ❌ Kan detektere gennem vinduer |

**Bedst til:** Stue, soveværelse, kontor, badeværelse - steder hvor du opholder dig.

## 🥇 Aqara FP2 - Bedste Plug-and-Play

<Tabs>
  <TabItem label="Oversigt">
    ### Hvorfor Aqara FP2?
    
    **Pris:** ~500 kr / $85
    
    Aqara FP2 er den **mest brugervenlige** mmWave sensor:
    
    - ✅ Op til 30 tilpassede zoner
    - ✅ HomeKit, Alexa, Google Home native
    - ✅ Flot app med zone-tegning
    - ✅ Lux sensor indbygget
    - ✅ Home Assistant via HomeKit integration
    - ✅ Vægmontering eller loftmontering
    
    **Ulemper:**
    - ❌ Dyr
    - ❌ Kræver WiFi (ingen Zigbee)
    - ❌ Langsommere detektion end DIY (~3.6 sek)
  </TabItem>
  <TabItem label="Specs">
    ### Tekniske Specifikationer
    
    | Spec | Værdi |
    |------|-------|
    | Protokol | WiFi 2.4GHz |
    | Rækkevidde | 5 meter |
    | Vinkel | 120° horisontalt |
    | Zoner | Op til 30 |
    | Strøm | USB-C 5V/1A |
    | Integration | HomeKit, HA, Alexa, Google |
  </TabItem>
  <TabItem label="Installation">
    ### Opsætning
    
    1. **Download Aqara Home app**
    2. **Tilslut USB-C strøm**
    3. **Tilføj enhed** i appen
    4. **Placer sensoren** - væg eller loft
    5. **Tegn zoner** i appen
    
    ### Home Assistant
    
    FP2 integreres via **HomeKit Controller**:
    
    1. Indstillinger → Enheder & Tjenester
    2. Tilføj Integration → HomeKit Controller
    3. Scan QR-kode fra Aqara app
    4. Zoner vises som separate binary_sensors
    
    **Entiteter:**
    - binary_sensor.fp2_presence
    - binary_sensor.fp2_zone_1
    - binary_sensor.fp2_zone_2 (osv.)
    - sensor.fp2_illuminance
  </TabItem>
</Tabs>

**Køb:** [Aqara.com](https://aqara.com), [Amazon.de](https://amazon.de), [Elgiganten](https://elgiganten.dk)

## 🥈 Everything Presence Lite - Bedste ESPHome

<Tabs>
  <TabItem label="Oversigt">
    ### Hvorfor EP Lite?
    
    **Pris:** ~350 kr / $50
    
    Everything Presence Lite er det **bedste kompromis**:

    - ❌ Statisk detektion svagere end EP One
  </TabItem>
  <TabItem label="Specs">
    ### Tekniske Specifikationer
    
    | Spec | Værdi |
    |------|-------|
    | mmWave Sensor | HiLink LD2450 |
    | Protokol | WiFi/ESPHome |
    | Rækkevidde | 6 meter |
    | Vinkel | 60° |
    | Multi-target | Op til 3 personer |
    | Zoner | 3 stk |
    | Ekstra | Lux sensor, BLE proxy |
  </TabItem>
  <TabItem label="Installation">
    ### Opsætning
    
    1. **Tilslut USB-C strøm**
    2. **Forbind til WiFi** via browser (Chrome)
       - Gå til [web.esphome.io](https://web.esphome.io)
       - Vælg "Install Everything Presence Lite"
    3. **Enhed opdages automatisk** i Home Assistant
    4. **Konfigurer zoner** via HA dashboard
    
    ### Zone Konfiguration
    
    I Home Assistant får du kontroller til:
    - Max afstand
    - Zone 1/2/3 koordinater
    - Target tracking (X, Y, afstand, hastighed)
    
    ```yaml
    # Eksempel automation
    automation:
      - alias: "Lys tændes i stue-zone"
        trigger:
          - platform: state
            entity_id: binary_sensor.ep_lite_zone_1
            to: "on"
        action:
          - service: light.turn_on
            entity_id: light.stue
    ```
  </TabItem>
</Tabs>

## 🔧 DIY: LD2410 + ESP32

Den **mest fleksible og billigste** løsning er at bygge din egen sensor!

### Hvorfor DIY?

- ✅ Kun ~100 kr total
- ✅ Fuld kontrol over alle parametre
- ✅ Kan kombinere med andre sensorer
- ✅ Community ESPHome support
- ✅ Ingen cloud

### Komponenter

| Del | Pris ca. | Hvor |
|-----|----------|------|
| **HiLink LD2410C** | 50 kr | AliExpress |
| **ESP32 DevKit** | 40 kr | AliExpress |
| **USB kabel + adapter** | 20 kr | Haves ofte |
| **Dupont kabler** | 10 kr | AliExpress |
| **Total** | ~120 kr | |

### LD2410 Varianter

| Model | Interface | Bluetooth | Anbefalet |
|-------|-----------|-----------|-----------|
| **LD2410** | 1.25mm stik | ❌ | Sværere at lodde |
| **LD2410B** | 1.25mm stik | ✅ | God, men små pins |
| **LD2410C** | 2.54mm pins | ✅ | ⭐ Nemmest at bruge |

### Tilslutning

```
ESP32          LD2410C
------         -------
3.3V   ────►   VCC
GND    ────►   GND
GPIO16 ────►   TX
GPIO17 ────►   RX
```

### ESPHome Konfiguration

```yaml
esphome:
  name: presence-sensor-stue

esp32:
  board: esp32dev

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password

uart:
  tx_pin: GPIO17
  rx_pin: GPIO16
  baud_rate: 256000
  parity: NONE
  stop_bits: 1

ld2410:
  id: ld2410_sensor

binary_sensor:
  - platform: ld2410
    has_target:
      name: "Stue Tilstedeværelse"
    has_moving_target:
      name: "Stue Bevægelse"
    has_still_target:
      name: "Stue Stillestående"

sensor:
  - platform: ld2410
    moving_distance:
      name: "Bevægelse Afstand"
    still_distance:
      name: "Stillestående Afstand"
    moving_energy:
      name: "Bevægelse Energi"
    still_energy:
      name: "Stillestående Energi"
    detection_distance:
      name: "Detektions Afstand"
    light:
      name: "Lys Niveau"

number:
  - platform: ld2410
    timeout:
      name: "Timeout"
    max_move_distance_gate:
      name: "Max Bevægelse Gate"
    max_still_distance_gate:
      name: "Max Stillestående Gate"

switch:
  - platform: ld2410
    engineering_mode:
      name: "Engineering Mode"
    bluetooth:
      name: "Bluetooth"
```

## 🏠 Automations Eksempler

### Basis: Lys ved tilstedeværelse

```yaml
automation:
  - alias: "Stue lys ved tilstedeværelse"
    trigger:
      - platform: state
        entity_id: binary_sensor.stue_tilstedevaerelse
        to: "on"
    condition:
      - condition: numeric_state
        entity_id: sensor.stue_lux
        below: 50
    action:
      - service: light.turn_on
        entity_id: light.stue
        data:
          brightness_pct: 80

  - alias: "Stue lys sluk når tom"
    trigger:
      - platform: state
        entity_id: binary_sensor.stue_tilstedevaerelse
        to: "off"
        for:
          minutes: 5
    action:
      - service: light.turn_off
        entity_id: light.stue
```

### Avanceret: Zone-baseret

```yaml
automation:
  - alias: "Skrivebordslampe ved arbejde"
    trigger:
      - platform: state
        entity_id: binary_sensor.kontor_zone_skrivebord
        to: "on"
    action:
      - service: light.turn_on
        entity_id: light.skrivebordslampe

  - alias: "Fjernsynsbelysning"
    trigger:
      - platform: state
        entity_id: binary_sensor.stue_zone_sofa
        to: "on"
    condition:
      - condition: state
        entity_id: media_player.tv
        state: "playing"
    action:
      - service: light.turn_on
        entity_id: light.tv_baggrund
        data:
          brightness_pct: 30
```
