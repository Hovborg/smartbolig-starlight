# AI-news pipeline v3 og arkiv-regenerering — 13. juli 2026

Fuld gennemgang og forbedring af AI-nyhedssystemet: indhold, pipeline,
kilder, billeder og daglig automatik. Supplerer 2026-07-13-site-audit.md
(og lukker det audits backlog-punkter om nyhedsarkivets skabelon-stempler
og og:image-størrelser).

## Problemer fundet

1. **Latent pipeline-stop:** v2-skabelonen skrev "Læs den officielle
   kilde:" (lille k), men ai-news-validate.mjs krævede bogstaveligt
   `Kilde:`/`Source:`. Næste rigtige nyhedsdag ville den daglige kørsel
   have fejlet på sin egen valideringsgate.
2. **Skabelon-stempling i hele arkivet:** alle 59 publicerede dage brugte
   den gamle skabelon med identiske sætninger i hver historie
   ("Det er en officiel kilde med praktisk betydning..." ×6 pr. dag).
   Selv v2-skabelonen genbrugte 4 faste analysesætninger på tværs af dage.
3. **8 gentagelses-dage** (ikke kun de 3 byte-identiske fra auditten):
   05-08, 05-17, 05-18, 06-07, 07-03, 07-04, 07-10 og 07-12 havde
   identiske kildesæt med foregående udgave, solgt som friske nyheder.
4. **Anthropic manglede som kilde** — "AI-nyheder" dækkede OpenAI og
   Google, men Anthropic var kun repræsenteret via Claude Code-releases.
5. **381 MB PNG-billeder** (op til 2,6 MB pr. og:image-variant).

## Ændringer

- **v3-renderer** (scripts/lib/ai-news-render.mjs): eksplicitte
  `Kilde:`/`Source:`-labels, signal high/medium/low efter kildetal,
  `copySource`-felt i frontmatter, ærlig `renderRepeatIssue` til dage
  uden nye kilder (signal: low + `repeatOf`).
- **LLM-redaktionslag** (scripts/lib/ai-news-llm.mjs): headless Claude
  Code (`claude -p`, model via AI_NEWS_LLM_MODEL) skriver unik
  per-historie-analyse (hvad/hvorfor/verificér/usikkerhed + lede) på
  dansk og engelsk ud fra hentet kildetekst. Kildemateriale behandles
  eksplicit som utroværdig data (prompt-injection-værn), output valideres
  hårdt (JSON-form, ordgrænser, URL/markdown/HTML-forbud) og escapes
  gennem samme safeText som feed-tekst. Enhver fejl → deterministisk
  skabelon-fallback; publicering kan aldrig blokere på LLM-laget.
- **Anthropic News som kilde**: www.anthropic.com/news har ingen RSS;
  ny HTML-listing-parser i discovery (+ source-health-understøttelse).
  Ikke-kritisk, så et markup-skifte ikke vælter kørslen.
- **Billeder**: alle hero/og-varianter genereres og serveres nu som JPEG
  (mozjpeg q82). Arkivet konverteret: 381 MB → 27 MB (~93 % mindre).
  Forsidens thumbs genereret fra JPEG-kilderne.
- **Arkivet regenereret**: alle 59 dage × 2 sprog via
  scripts/ai-news-regenerate.mjs (genopbygger fra hver artikels egen
  kildetabel, henter kilderne igen for materiale, LLM-copy pr. dag,
  resumérbar). Resultat: 51 LLM-dage + 8 ærlige gentagelses-udgaver,
  0 skabelon-fallbacks.
- **Automatik**: daglig kørsel sætter AI_NEWS_LLM=1 (kan slås fra);
  PR-teksten viser copy-kilden. Content-skemaet udvidet med
  editorialVersion 3, copySource og repeatOf.
